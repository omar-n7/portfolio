06

Databases

Having the ability to manage back-end information using databases is key for any web developer and it is a tool that is guaranteed to come in handy when working with large scale websites that have a lot of information that needs to be stored and updated in a secure environment. I have experience using MySQL, MongoDB, and Postman which technically isn't a database application but it allows you to test your own APIs and use that information to update your databases. I built an e-commerce website using the MERN stack and Postman was very helpful in testing the back-end aspects of the application such as user creation, product creation/updates, and sending/receiving requests from the custom API. This is why I feel that all front end developers should have experience with databases and APIs in at least some sort of capacity.

07

WordPress

WordPress is a phenomenal tool for people who aren't experienced programmers and it's back-end interface is very easy to understand and use for non-technical users. I used WordPress to create an e-commerce website for a client because they wanted to be able to manage their business side of things through the website without having to do any programming and with the help of different plugins, it's extremely easy to not only manage and update your business information, but also change certain aspects of your website without any prior coding experience. All the client needs is a username and password and they're good to go. While the same thing can be accomplished through JavaScript and PHP, the client wanted the website built within a fairly short timeline so I decided to take advantage of the benefits of using WordPress to create an e-commerce website.
